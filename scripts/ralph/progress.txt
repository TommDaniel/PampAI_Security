## Codebase Patterns
- Alert pattern (webhook + email): add send_*_alert(org_id, event) to alerts.py; call both from auto-persist with asyncio.ensure_future(); guard with `if is_phishing and org_id`
- Email alert: uses Python built-in smtplib (no new deps); config via SMTP_HOST/PORT/USER/PASSWORD/FROM env vars; `endpoint` in alert_configs is the recipient email address
- Webhook alert pattern: create alerts.py with send_webhook_alert(org_id, event); call from auto-persist with asyncio.ensure_future(); guard with `if is_phishing and org_id`
- Extension auth header pattern: getAuthHeaders() returns {X-API-Key, X-User-Email} from cached identity; spread into fetch headers; returns {} if no identity configured (anonymous requests still work)
- httpx.AsyncClient is available (in requirements.txt); use for async HTTP in alerts system
- phishing-api/app.py uses Pydantic BaseModel + Field for all request/response models
- API is FastAPI (not Flask), running on Uvicorn; lifespan context manager handles startup/shutdown
- DB connection uses SQLAlchemy async (create_async_engine + async_sessionmaker) with asyncpg driver
- DATABASE_URL env var: `postgresql+asyncpg://user:pass@host:5432/db` — defaults to localhost
- DB_ENABLED flag in db.py: init_db() sets True only on successful connect (graceful degradation)
- Migrations SQL in phishing-api/migrations/ — SQLAlchemy also creates tables via metadata.create_all
- Python syntax check: `python3 -c "import ast; ast.parse(open('file').read())"`
- No pip available in devcontainer — cannot install Python packages for runtime testing
- Auth dependency `get_org_id` in auth.py: returns None (anonymous) or org_id string; raises 401 for bad key, 503 if DB down
- Admin org creation: POST /admin/orgs (no auth required for bootstrap); returns api_key
- Auth is OPTIONAL on existing endpoints — anonymous requests (no X-API-Key) are allowed
- log_event() in db.py uses `.returning(id, created_at)` — merge with values dict for full row; created_at is a Python datetime, call .isoformat() for JSON
- Patch `db.log_event` in tests (not `app.log_event`) — endpoint uses `from db import log_event` inside handler
- Auto-persist pattern: guard `if DB_ENABLED:`, call `await log_event(...)`, wrap in try/except — never fail the response
- Batch auto-persist: use `asyncio.ensure_future(_persist_batch())` to avoid blocking response
- Reports endpoint: require auth (401 if no key), then 403 if caller_org_id != path org_id; use dependency_overrides in tests
- SQLAlchemy conditional counts: `func.count(case((table.c.col == val, 1)))` — counts non-null results (requires `case` imported from sqlalchemy)
- Test auth overrides: `app.dependency_overrides[get_org_id] = lambda: async_fn()` — always clear after test with `app.dependency_overrides.clear()`
