## Codebase Patterns
- phishing-api/app.py uses Pydantic BaseModel + Field for all request/response models
- Working tree has CRLF line endings on many files (.gitattributes controls this)
- Extension lives in extensao-phishing/, API in phishing-api/
- TypeScript typecheck: `npx tsc --noEmit` from extensao-phishing/
- Python syntax check: `python3 -c "import ast; ast.parse(open('file').read())"`
- No pip available in devcontainer - cannot install Python packages for runtime testing
- Existing cascade architecture: BERT (URL) -> CatBoost (WHOIS/DNS) with confidence thresholds
- Email/translation models load with try/except so API still starts if HuggingFace models unavailable
- MarianMT uses MarianTokenizer + MarianMTModel from transformers (not AutoTokenizer)
- Email tokenizer uses max_length=512 (not 128 like URL tokenizer) — tests should not assert 512 is absent from source
- Test helpers: `_create_mock_model()` and `_create_mock_tokenizer()` for reusable mock creation
